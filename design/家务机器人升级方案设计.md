# 家务机器人升级方案设计（轮子 + 升降台 + Stewart + 双臂）

## 1. 目标与边界
- 目标场景：室内家务（平地、低门槛、长时运行）。
- 形态目标：保留双臂与轮式移动，增加中间机构实现“身高变化 + 姿态变化”。
- 不追求：双腿步态、跨大障碍、重载搬运。

### 1.1 关键指标（V1）
- 连续工作时长：8 小时。
- 视角/操作高度等效：覆盖约 1.5m~1.9m 人体工作区间（通过升降实现）。
- 单手常用负载：1~3kg；短时峰值：5kg（10kg 作为远期目标，不纳入 V1）。
- 地形适应：室内平整地面、小门槛/地毯可通过。

## 2. 硬件定义

## 2.1 机械分层
- L0 底盘层：双轮差速底盘（已具备）。
- L1 垂向层：电动升降台（大行程、低频）。
- L2 姿态层：Stewart Platform（6-DOF，小行程、高带宽）。
- L3 操作层：双臂与头部相机系统（已具备）。

## 2.2 建议参数（V1）
- 升降台行程：250~350mm（优先 300mm）。
- Stewart 姿态范围（先保守）：
  - roll: ±15°
  - pitch: -20° ~ +25°
  - yaw: ±20°
- Stewart 平移范围（用于微调）：
  - z: ±30~50mm
  - x/y: 先限制在 ±20mm 内

## 2.3 传感与执行
- 必需传感器：
  - IMU（机体姿态闭环）
  - 升降台编码器/位置传感
  - Stewart 六执行器位置反馈
  - 电池电压/电流/温度
- 必需执行器：
  - 升降台电机驱动（带软硬限位）
  - Stewart 六执行器驱动（带行程保护）

## 2.4 供电与续航
- 续航估算：`电池 Wh = 平均功耗(W) * 8h * 1.2`
- 建议做法：
  - 先实测三种工况平均功耗：待机、移动、操作+姿态联动
  - 以“中位工况”反推电池规格

## 3. 软件接口定义

## 3.1 控制总线分层
- 任务层（Task）：高层动作/任务规划。
- 协调层（Coordinator）：底盘、升降台、Stewart、双臂的目标分配。
- 执行层（Driver）：各硬件驱动与安全保护。

## 3.2 统一命令接口（建议）
- `body/height_cmd`（float, m）
  - 含义：机体目标高度（由升降台主执行）
- `body/attitude_cmd`（roll, pitch, yaw, rad）
  - 含义：机体目标姿态（由 Stewart 主执行）
- `body/pose_trim_cmd`（x, y, z, m）
  - 含义：小范围平移微调（Stewart）
- `base/vel_cmd`（vx, wz）
  - 含义：底盘速度命令
- `arm/left_cmd`、`arm/right_cmd`
  - 含义：双臂动作命令（关节或末端）
- `safety/estop`
  - 含义：急停（全系统最高优先级）

## 3.3 状态反馈接口（建议）
- `body/state`
  - 当前高度、姿态、Stewart 各执行器状态
- `base/state`
  - 底盘里程计、速度、电机状态
- `arm/state`
  - 双臂关节状态、夹爪状态
- `power/state`
  - 电池 SoC、电压、电流、温度、剩余续航估算
- `system/health`
  - 错误码、告警码、看门狗状态

## 3.4 安全约束接口
- `limits/body`
  - 高度上限/下限、姿态上限、速度上限
- `limits/arm`
  - 双臂速度/加速度限制、与姿态联动限幅
- `limits/base`
  - 转弯速度随姿态/负载动态降额

## 4. 仿真定义

## 4.1 仿真目标
- 验证运动学可达性：升降 + Stewart + 双臂无干涉。
- 验证稳定性：底盘运动时姿态补偿有效。
- 验证任务可行性：典型家务动作在目标高度范围内可完成。

## 4.2 模型与环境
- 物理引擎：优先使用现有 MuJoCo/Isaac 体系中的一种（保持当前工程一致）。
- 模型要求：
  - 底盘差速模型
  - 升降台 1-DOF 模型
  - Stewart 6-DOF 机构模型（执行器长度约束）
  - 双臂与相机外参

## 4.3 仿真测试项（V1）
- T1：升降台阶跃响应（上升/下降时间、超调、稳态误差）。
- T2：Stewart 姿态跟踪（roll/pitch/yaw 正弦扫描）。
- T3：底盘加减速 + 姿态补偿（俯仰抑制效果）。
- T4：双臂操作时重心扰动 + 平台补偿。
- T5：极限保护测试（限位、急停、失联回中）。

## 5. 算法定义

## 5.1 运动学与分配
- 升降台：
  - 高度主通道控制，低频大行程。
- Stewart：
  - 姿态主通道 + 小平移微调。
  - 输入：目标姿态/微位移；输出：6 执行器目标长度。
  - 需带执行器限位与奇异位形规避。

## 5.2 协调控制策略（核心）
- 规则 R1：`z` 高度优先由升降台完成，Stewart 只做微调。
- 规则 R2：`roll/pitch/yaw` 优先由 Stewart 完成，升降台不参与姿态闭环。
- 规则 R3：底盘高速/急转时自动降低双臂速度上限。
- 规则 R4：双臂大负载动作时，底盘速度降额，Stewart 提高补偿增益。

## 5.3 控制频率建议
- 底盘与 Stewart：50~100Hz
- 升降台：20~50Hz
- 双臂：30~50Hz
- 状态估计与安全监测：100Hz

## 5.4 状态机（建议）
- `IDLE`：待命
- `MOBILE`：底盘移动
- `MANIPULATION`：双臂操作
- `POSTURE_COMP`：姿态增强补偿
- `E_STOP`：急停
- `FAULT_RECOVERY`：故障恢复（回中位/限速/重新上电）

## 5.5 安全算法
- 看门狗：控制指令超时立即停车 + 平台回中。
- 限位器：高度、姿态、执行器长度三重限幅。
- 负载降额：检测高电流/高温时动态降速。
- 倾覆保护：姿态越界触发“降重心 + 降速 + 停止大幅动作”。

## 6. 实施里程碑
- M1（2~4 周）：升降台单独闭环 + 安全限位。
- M2（2~4 周）：Stewart 单独控制 + IK + 限位保护。
- M3（3~6 周）：与底盘联动补偿（不带双臂任务）。
- M4（3~6 周）：双臂联动 + 家务任务验证（抓取/放置/台面操作）。
- M5（持续迭代）：能耗优化与 8 小时续航达标。

## 7. 验收标准（V1）
- 能完成“移动到位 -> 调高/调姿 -> 双臂操作 -> 安全返回”闭环任务。
- 连续运行 8 小时内无致命故障（允许可恢复告警）。
- 常见家务任务下姿态补偿有效，操作成功率达到目标阈值（由任务定义给出）。

## 8. Jetson + Arduino 控制分层（推荐）

## 8.1 分层职责
- Jetson（高层）：
  - 视觉、任务、策略推理（ACT）
  - 输出 `height_cmd`、`attitude_cmd`、`estop`
  - 建议发送频率：50~100Hz
- Arduino/MCU（低层实时）：
  - Stewart 六执行器闭环（位置/速度）
  - 升降台位置闭环
  - 限位、急停、通信超时回中
  - 建议控制频率：200~500Hz

## 8.2 硬件建议
- UNO 不建议（算力和外设余量偏小）。
- 推荐：Teensy 4.x / STM32 / ESP32（按驱动接口选择）。
- 原则：低层控制器必须独立于 Jetson 具备安全能力（失联可进入安全模式）。

## 9. Jetson-MCU 通信协议（最小版）

## 9.1 帧格式（小端）
- Header: `0xAA 0x55`
- `msg_type` (u8)
- `seq` (u16)
- `payload_len` (u16)
- `payload` (bytes)
- `crc16` (u16, CCITT-FALSE)

## 9.2 消息定义
- `0x01 CMD_BODY`
  - `height_m` (float32)
  - `roll_rad` (float32)
  - `pitch_rad` (float32)
  - `yaw_rad` (float32)
  - `flags` (u8) bit0=`estop`
- `0x02 CMD_ENABLE`
  - `enable` (u8)
- `0x81 STATE_BODY`
  - `height_m` (float32)
  - `roll_rad` (float32)
  - `pitch_rad` (float32)
  - `yaw_rad` (float32)
  - `fault_code` (u16)
- `0x82 STATE_ACTUATOR`
  - `lift_pos` (float32)
  - `stewart_len[6]` (float32 * 6)
  - `stewart_current[6]` (float32 * 6)

## 9.3 超时与安全策略
- 通信超时：`>100ms` 无新 `CMD_BODY` 则进入 `SAFE_HOLD`：
  - 升降台停止并缓慢降到安全高度
  - Stewart 回中（姿态 0）
  - 禁止大幅动作，等待人工恢复
- 急停：任何时刻 `flags.estop=1` 立即切断执行输出。

## 10. 参考代码骨架

## 10.1 Arduino/MCU 侧（伪代码）
```cpp
// loop @ 500Hz
void loop() {
  readSerialFrames();
  if (estop) {
    disableAllActuators();
    return;
  }
  if (millis() - last_cmd_ms > 100) {
    enterSafeHold();  // center stewart + stop/slow lower lift
  }

  // IK: attitude + trim -> leg targets
  computeStewartTargets(cmd_roll, cmd_pitch, cmd_yaw, cmd_z_trim);

  // Closed-loop control
  updateLiftPID(cmd_height, measured_height);
  for (int i = 0; i < 6; ++i) {
    updateLegPID(i, target_leg_len[i], measured_leg_len[i]);
  }

  enforceLimits();   // stroke/velocity/current limits
  publishState();    // STATE_BODY / STATE_ACTUATOR
}
```

## 10.2 Jetson Python 侧（最小发送示例）
```python
import struct
import serial
import time

def crc16_ccitt(data: bytes) -> int:
    crc = 0xFFFF
    for b in data:
        crc ^= (b << 8)
        for _ in range(8):
            crc = ((crc << 1) ^ 0x1021) & 0xFFFF if (crc & 0x8000) else ((crc << 1) & 0xFFFF)
    return crc

def pack_cmd_body(seq, h, r, p, y, estop=False):
    msg_type = 0x01
    flags = 0x01 if estop else 0x00
    payload = struct.pack("<ffffB", h, r, p, y, flags)
    head = struct.pack("<2sBHH", b"\xAA\x55", msg_type, seq, len(payload))
    crc = crc16_ccitt(head + payload)
    return head + payload + struct.pack("<H", crc)

ser = serial.Serial("/dev/ttyACM0", 115200, timeout=0.01)
seq = 0
while True:
    frame = pack_cmd_body(seq, h=0.18, r=0.0, p=0.0, y=0.0, estop=False)
    ser.write(frame)
    seq = (seq + 1) & 0xFFFF
    time.sleep(0.02)  # 50Hz
```

## 11. 集成顺序（执行建议）
- S1：仅开升降台闭环（固定姿态，禁用 Stewart）。
- S2：仅开 Stewart 的 roll/pitch（小角度 ±5°）。
- S3：升降 + Stewart 联动（不接双臂）。
- S4：接入双臂，开启动态降额与倾覆保护。
