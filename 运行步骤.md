# XLeRobot 运行步骤（固定使用 nvidia 环境）

## 0. 约定（重要）
- 保留并只使用 `nvidia` 环境：`/home/nvidia/.venv/bin/python`
- 不使用 `lerobot-main/.venv/bin/python`
- 所有命令都在 `~/Desktop/lerobot-main` 目录执行

## 1. 先验证当前解释器
```bash
cd /home/nvidia/Desktop/lerobot-main
/home/nvidia/.venv/bin/python --version
/home/nvidia/.venv/bin/python -c "import lerobot; print(lerobot.__file__)"
```

预期：
- Python 版本是 `3.10.12`（或你的 nvidia venv 实际版本）
- `lerobot` 路径指向 `/home/nvidia/Desktop/lerobot-main/src/lerobot/__init__.py`

## 2. 软件侧自检（不接机械臂/手柄）

## 3. 启动 xlerobot host（未接硬件时）
```bash
cd /home/nvidia/Desktop/lerobot-main
/home/nvidia/.venv/bin/python -m lerobot.robots.xlerobot.xlerobot_host --robot.id=my_xlerobot
```

如果出现下面错误，属于**正常现象**（当前未连接控制板）：
- `Could not connect on port '/dev/ttyACM0'`
- `No such file or directory: '/dev/ttyACM0'`

这代表：软件已经跑通到“串口连接硬件”这一步。

## 4. 接上机械臂后要做的事
### 4.1 查看真实串口
```bash

```

文件：

参数：
- `port1`
- `port2`

改成实际设备端口（例如 `/dev/ttyACM1`、`/dev/ttyUSB0`）。

### 4.3 再次启动
```bash

/home/nvidia/.venv/bin/python -m lerobot.robots.xlerobot.xlerobot_host --robot.id=my_xlerobot
```

## 5. 常见问题
- 现象：`ModuleNotFoundError: No module named 'lerobot'`
  - 原因：用了错误环境（通常是 `lerobot-main/.venv`）
  - 处理：改用 `/home/nvidia/.venv/bin/python`

- 现象：命令行显示 `(nvidia) (base)`
  - 说明：conda base 与 venv 提示同时存在，一般不影响
  - 关键：命令必须显式使用 `/home/nvidia/.venv/bin/python`

- 现象：手柄未连接时 `pygame.joystick.get_count()==0`
  - 说明：正常

## 9. 自己采集并训练 ACT（推荐）
> 用别人的模型可以试流程，但实机效果通常不稳定。推荐采集自己的数据并训练。

### 9.1 数据采集（单臂，最容易先跑通）
```bash

  --robot.type=so101_follower \
  --robot.port=<follower_port> \
  --robot.id=robot_right_arm \
  --robot.cameras="{ head: {type: opencv, index_or_path: <cam_head_idx>, width: 640, height: 480, fps: 20} }" \
  --teleop.type=so101_leader \
  --teleop.port=<leader_port> \
  --teleop.id=my_leader_arm \
  --display_data=true \
  --dataset.repo_id=<你的HF用户名>/xlerobot_single_arm_task \
  --dataset.num_episodes=50 \
  --dataset.single_task="your task"
```

### 9.2 数据采集（双臂+轮子，按 VLA_ACT 的 VR 流程）
```bash
cd /home/nvidia/Desktop/lerobot-main

  --robot.type=xlerobot \
  --robot.cameras="{ head: {type: opencv, index_or_path: <cam_head_idx>, width: 640, height: 480, fps: 20}, right: {type: opencv, index_or_path: <cam_right_idx>, width: 640, height: 480, fps: 20}, left: {type: opencv, index_or_path: <cam_left_idx>, width: 640, height: 480, fps: 20} }" \
  --dataset.repo_id=<你的HF用户名>/xlerobot_bimanual_task \
  --dataset.single_task="your task" \
  --dataset.root=/home/nvidia/Desktop/datasets/xlerobot_bimanual_task \
  --display_data=true \
  --teleop.type=xlerobot_vr
```

### 9.3 训练 ACT（用自己的数据）
```bash
cd /home/nvidia/Desktop/lerobot-main
py310nv -m lerobot.scripts.lerobot_train \
  --dataset.repo_id=<你的HF用户名>/xlerobot_bimanual_task \

  --batch_size=64 \

  --output_dir=/home/nvidia/Desktop/lerobot-main/outputs/act_xlerobot_bimanual_task
```

## 10. 训练环境要求（CPU/GPU）与建议
### 10.1 为什么建议在 PC 训练
- ACT 可以在机器人端做推理，但训练更吃算力与显存。
- 文档给出的量级：ACT 约 `~80M` 参数，单卡训练可在数小时内完成。
- 文档示例提到：`80k steps` 在 `NVIDIA A100` 约 `1h45`。

### 10.2 推荐硬件（训练）
- GPU：
  - 最低可用：`NVIDIA 12GB` 显存（需要减小 batch、速度较慢）
  - 推荐：`NVIDIA 24GB` 显存（更稳）
  - 理想：`A100/4090` 级别（训练时间明显缩短）
- CPU：`8` 核及以上
- 内存：`32GB` 起步，推荐 `64GB`
- 存储：NVMe SSD，至少预留 `100GB`（数据集 + checkpoint + 缓存）

### 10.3 Jetson 的定位
- 推荐用于：机械臂在线推理、数据采集、部署验证。
- 不推荐用于：长时间大规模训练（效率低、调参周期长）。

## 11. 本机当前探测结果（2026-02-27）
- `ls /dev/ttyACM* /dev/ttyUSB*`：当前未检测到可用设备（你应在连接机械臂后重试）。
- `lerobot_find_cameras opencv`：当前未检测到 OpenCV 相机（接好相机后重试）。
- `lerobot-find-port`：是交互式工具，需要按提示“拔插 USB 后按回车”才能返回具体端口。
